OAuth User Flow
Step-by-step PKCE flow to get a User Access Token

Step 1 — Generate Auth URL

Generate the authorization URL with PKCE. The user will be redirected here to grant permissions.

Step 2 — Handle Callback

After the user authorizes, Blaze redirects back with a code and state. Paste the full callback URL or just the code here.

Step 3 — Refresh Token

Use the refresh token to get a new access token without re-auth.

Login with Blaze (User OAuth)
One-click PKCE flow — click the button in the header or below. After you approve on Blaze, you'll be redirected back here and the token auto-fills.

Step 1 — Click Login

Hit the button below. We call Blaze to generate an auth URL (PKCE), save the state + verifier in sessionStorage, then redirect you to Blaze's login page.

Step 2 — Approve on Blaze

You'll see a Blaze screen asking to authorize this app with scopes: users.read offline.access channel.moderate users.bot. Click Approve.

Step 3 — Auto Callback

Blaze redirects you back here with a ?code= in the URL. The page detects it automatically, exchanges the code for a User Access Token, and fills it in the auth bar above. No manual pasting needed.
